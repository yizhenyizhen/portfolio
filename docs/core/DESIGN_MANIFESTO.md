# DESIGN_MANIFESTO.md

---

# 1. Purpose

This document defines the visual and emotional language of the Yizhen Zhou website.

Unlike the Product Bible, which defines what the product is, this document defines how the product should feel.

Every design decision should be evaluated against this manifesto before implementation.

If a design looks impressive but violates this document, it should be rejected.

---

# 2. Design Philosophy

The website should never try to impress people.

It should quietly communicate confidence.

Confidence does not require explanation.

Confidence is visible through consistency.

Visitors should feel that every pixel has a reason to exist.

Nothing should feel accidental.

Nothing should feel excessive.

Nothing should exist simply because it looks cool.

---

# 3. Design Identity

The visual identity can be summarized in six words.

Calm.

Precise.

Intentional.

Timeless.

Interactive.

Human.

Every future page should naturally inherit these characteristics.

If a page feels louder than these words, it is probably wrong.

---

# 4. Atmosphere Before Aesthetics

Most designers focus on appearance.

This project focuses on atmosphere.

Visitors should remember the emotional feeling of the website before remembering individual visual elements.

The objective is not visual stimulation.

The objective is emotional memory.

When visitors leave, they should remember how the website felt.

Not which font was used.

Not which animation was used.

Not which color was used.

---

# 5. Restraint

The highest form of confidence is restraint.

Do not fill empty space.

Do not explain everything.

Do not animate everything.

Do not highlight everything.

Leave room for curiosity.

Leave room for imagination.

Silence creates attention.

Space creates hierarchy.

Restraint creates premium experiences.

---

# 6. Simplicity

Simple does not mean easy.

Simple means every unnecessary element has already been removed.

Visitors should never think.

"This looks minimal."

Instead they should think.

"This feels effortless."

Effortlessness is the result of careful decisions.

Not the absence of decisions.

---

# 7. Timelessness

Design trends change every year.

This website should not.

Avoid visual styles that immediately reveal a specific design era.

The website should still feel contemporary after many years.

Long-term identity is more important than short-term attention.

---

# 8. Interaction

Interaction is communication.

Every interaction should answer one question.

Why does this exist?

If there is no meaningful answer, remove it.

Interactions should feel natural.

Not surprising.

Not dramatic.

Not theatrical.

Motion should reveal information.

Never distract from information.

---

# 9. Premium

Premium is not expensive.

Premium is not luxury.

Premium is clarity.

Premium is confidence.

Premium is consistency.

Premium experiences remove friction instead of adding decoration.

Premium products respect the user's attention.

This website should do the same.

---

# 10. Visual Hierarchy

The website should never compete for attention.

Only one element should dominate the screen at any given moment.

Everything else should support that element.

Visitors should always know where to look without consciously thinking about it.

Hierarchy should be created through spacing, scale, timing and contrast.

Never rely on bright colors to attract attention.

Never rely on excessive animation.

The interface should guide the eye naturally.

---

# 11. Typography

Typography should feel architectural.

Words should behave like materials.

Large titles create structure.

Small text provides support.

Body text should remain short.

Whenever possible, replace explanation with interaction.

Paragraphs should be rare.

Most ideas should be communicated using one sentence.

One clear sentence is stronger than ten average ones.

Reading should never become work.

---

# 12. Color

Black is the foundation.

White provides clarity.

Accent colors exist only to communicate hierarchy.

Accent colors should never become decoration.

The interface should still feel complete even if the accent color is removed.

Future accent colors must remain restrained.

Avoid gradients unless they communicate depth.

Avoid colorful interfaces.

The website should feel calm before it feels beautiful.

---

# 13. Materials

Digital interfaces should borrow inspiration from physical materials.

Metal.

Glass.

Concrete.

Paper.

Shadow.

Light.

These materials should influence behavior instead of appearance.

Interfaces should feel solid.

Motion should have weight.

Depth should feel believable.

Nothing should feel artificial.

---

# 14. Space

Empty space is not unused space.

Empty space creates rhythm.

Empty space creates confidence.

Do not reduce spacing simply to fit more information.

The website should breathe.

Visitors should never feel crowded.

Every screen should feel intentionally composed.

---

# 15. Images

Images are not decoration.

Every image should communicate something unavailable through text.

Image quality is non-negotiable.

Low-resolution images should never appear.

If an image does not improve understanding, remove it.

Photography should feel cinematic rather than documentary.

Architectural images should emphasize atmosphere before information.

---

# 16. Motion

Motion should feel mechanical rather than playful.

Acceleration should feel natural.

Deceleration should feel intentional.

Transitions should communicate movement through space rather than fading between screens.

Motion should create continuity.

Visitors should always understand where they came from and where they are going.

Fast animations create excitement.

Slow animations create confidence.

This project prefers confidence.

---

# 17. Sound

The website should remain fully functional without sound.

If sound is ever introduced, it must be optional.

Sound should reinforce atmosphere.

Never demand attention.

Silence should remain the default experience.

---

# 18. Scrolling

Scrolling should tell stories.

Scrolling should never simply reveal more content.

Each scroll should feel like progressing through an experience.

Large sections should unfold gradually.

Visitors should never feel they are reading one endless page.

Instead, they should feel they are moving through different spaces.

---

# 19. What Feels Premium

Premium experiences are rarely complicated.

They are carefully edited.

The feeling of quality comes from decisions rather than decoration.

Premium design is built through consistency.

Every spacing value should feel intentional.

Every animation should have a purpose.

Every transition should preserve continuity.

Every page should feel like it belongs to the same product.

Premium experiences respect attention.

They never compete with themselves.

They never ask users to look everywhere at once.

They quietly guide focus.

Premium experiences reward exploration.

The more time visitors spend, the more details they discover.

The experience becomes richer without becoming more complicated.

---

# 20. What Feels Cheap

The following characteristics should never appear.

Large blocks of unnecessary text.

Template layouts.

Overly colorful interfaces.

Excessive glassmorphism.

Animations that exist only to attract attention.

Random gradients.

Inconsistent spacing.

Multiple visual styles on the same page.

Generic stock photography.

Decorative icons without meaning.

Rounded cards everywhere.

Corporate call-to-action sections.

Pages that immediately reveal every piece of information.

Design should never feel like software generated from a template.

Visitors should never recognise a UI library before they recognise Yizhen Zhou.

---

# 21. Inspiration

Inspiration is encouraged.

Copying is forbidden.

Reference products should be studied for their thinking rather than their appearance.

Apple demonstrates clarity.

Raycast demonstrates confidence.

Nothing demonstrates personality.

Aman demonstrates atmosphere.

TradingView demonstrates information hierarchy.

Rick Owens demonstrates presentation.

Bartlett Summer Show demonstrates storytelling.

These references define principles.

They do not define visual outcomes.

The final website should not remind visitors of any one reference.

Instead, it should feel like a natural product of Yizhen Zhou.

---

# 22. Steven Style

The website should reflect the personality of its creator.

Creative without chaos.

Disciplined without rigidity.

Minimal without emptiness.

Premium without luxury.

Technical without intimidation.

Interactive without becoming a game.

Visitors should feel they are meeting someone who enjoys building systems, refining details and continuously improving life.

The website should quietly communicate ambition.

It should never announce ambition.

---

# 23. Storytelling

Stories should unfold naturally.

Do not force visitors through linear narratives.

Allow exploration.

Allow discovery.

Allow different visitors to create different journeys.

Architecture students may begin with Portfolio.

Friends may begin with Running.

Hotel enthusiasts may begin with Room Keys.

Future collaborators may begin with AI.

Every entry point should eventually lead to the same understanding.

Different paths.

One identity.

---

# 24. Emotional Consistency

Every page should create the same emotional atmosphere.

Changing content should not change personality.

Visitors should never feel they have entered another website.

Transitions between sections should feel like walking into another room inside the same house.

The architecture changes.

The identity remains.

---

# 25. Attention

Attention is the most valuable resource visitors give.

Never waste it.

Do not interrupt.

Do not overload.

Do not repeat.

Remove everything that does not increase understanding.

Whenever a design decision becomes difficult, choose the option that respects the visitor's attention.

---

# 26. Non-Negotiable Rules

The following rules define the identity of this project.

They should remain true regardless of future technologies, design trends or business goals.

Breaking one of these rules requires changing the philosophy of the product itself.

---

## Rule One

Curiosity always comes before explanation.

Visitors should want to know more before they are told more.

---

## Rule Two

Experience always comes before interface.

Interfaces exist to deliver experiences.

Experiences create memory.

Memory creates identity.

---

## Rule Three

The website should never feel finished.

It should feel alive.

Every visit should suggest that more stories are waiting to be discovered.

---

## Rule Four

Every interaction must communicate meaning.

Motion without communication is decoration.

Decoration without purpose is noise.

---

## Rule Five

Visitors should discover Yizhen Zhou instead of reading about Yizhen Zhou.

Discovery creates stronger emotional connections than description.

---

## Rule Six

Remove before adding.

Whenever a new feature is proposed, first ask what can be removed.

Complexity grows naturally.

Simplicity requires discipline.

---

## Rule Seven

Every screen should have only one primary focus.

When everything competes for attention, nothing is remembered.

---

## Rule Eight

Technology should never become the protagonist.

Visitors should remember the experience.

Not React.

Not Three.js.

Not AI.

Technology is invisible when it succeeds.

---

## Rule Nine

Every future chapter must naturally belong to the same life story.

Growth should feel inevitable.

Never random.

---

## Rule Ten

Build slowly.

Build carefully.

Build for the next ten years.

Never build for next week's trend.

---

# 27. Design Review Checklist

Every page should pass the following review before release.

---

## Identity

Does this page feel uniquely Yizhen Zhou?

Could this page belong to someone else?

If yes, redesign it.

---

## Simplicity

Can anything be removed?

If removing something improves clarity, remove it.

---

## Emotion

Does the page create curiosity?

Or does it simply provide information?

Curiosity should always win.

---

## Hierarchy

Is there one obvious visual focus?

If visitors do not immediately know where to look, improve the hierarchy.

---

## Motion

If every animation disappears, is the page still beautiful?

If not, redesign the page.

Motion should enhance quality.

Never replace quality.

---

## Content

Can the same idea be communicated using fewer words?

If yes, rewrite it.

---

## Interaction

Does every interactive element explain itself naturally?

Visitors should never require instructions.

---

## Consistency

Does this page feel like another room inside the same building?

Or does it feel like another website?

If it feels different, redesign it.

---

## Timelessness

Will this page still feel contemporary five years from now?

If the answer is uncertain, simplify it.

---

## Memory

After closing the page, what will visitors remember?

If the answer is "animation" or "layout", the page has failed.

If the answer is "Yizhen Zhou", the page has succeeded.

---

# 28. Closing Statement

This manifesto is not a collection of design rules.

It is a way of thinking.

Every future decision should begin here.

When uncertainty appears, simplify.

When complexity appears, remove.

When trends appear, ignore them.

When inspiration appears, understand it instead of copying it.

The purpose of this website is not to become the most impressive personal website on the Internet.

The purpose is to become the most authentic digital representation of one person.

As long as visitors leave with a clearer understanding of who Yizhen Zhou is, every design decision has achieved its purpose.

Design should never speak louder than the person behind it.

End of Design Manifesto.
