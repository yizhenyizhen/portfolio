# HOME_PRD.md

---

# 1. Purpose

The homepage is the first impression of the entire website.

Its purpose is not to introduce Yizhen Zhou.

Its purpose is to create curiosity.

Visitors should immediately realise that this is an interactive experience instead of a traditional portfolio website.

The homepage should encourage exploration before explanation.

---

# 2. Success Criteria

The homepage succeeds if visitors naturally interact with the Ring within the first few seconds.

The homepage succeeds if visitors become curious enough to enter another section.

The homepage succeeds if visitors remember the interaction after leaving the website.

The homepage fails if visitors immediately scroll looking for information.

The homepage fails if visitors believe they are reading a normal portfolio.

---

# 3. First Impression

The first screen should feel calm.

There should be no visual noise.

The background should remain almost entirely black.

Typography should remain minimal.

The Ring should become the visual focus.

Visitors should immediately understand that interaction is possible.

No instructions should be displayed unless absolutely necessary.

The interaction should teach itself.

---

# 4. Homepage Layout

The homepage contains four permanent layers.

Layer 1

Background

Layer 2

Ambient visual effects

Layer 3

Ring Navigation

Layer 4

Minimal interface elements

The Ring should always remain the visual center.

Everything else supports it.

Nothing competes with it.

---

# 5. Background

The background should remain fixed.

Scrolling should never move the background.

The visual language should reference Nothing's subtle dot texture without copying it.

Depth should come from light rather than decoration.

The background exists to create atmosphere.

Not attention.

---

# 6. Ring Navigation

The Ring is the identity of the homepage.

The Ring should appear suspended in space.

Items are evenly distributed around the circle.

The currently selected item is always visually dominant.

Unselected items should remain readable but secondary.

The Ring should feel mechanical.

Never playful.

Never cartoon-like.

Never game-like.

---

# 7. Mouse Interaction

Mouse interaction is the primary control method on desktop.

Visitors should immediately understand that the Ring can be controlled without reading instructions.

The mouse wheel rotates the Ring.

The cursor should remain simple.

No drag interaction is required in Version 1.

Future versions may support dragging.

The interaction should always feel predictable.

Every movement should feel intentional.

---

# 8. Scroll Behaviour

Scrolling controls the Ring.

Scrolling does not move the webpage.

Until a world has been selected, the page remains fixed.

The homepage behaves more like a product experience than a document.

Once a world is entered, traditional scrolling becomes available.

This creates a clear distinction between exploration and reading.

---

# 9. Ring Physics

The Ring should never rotate one step at a time.

Instead, it should rotate continuously with physical momentum.

The motion should feel inspired by:

Apple Watch Digital Crown.

Apple Vision Pro.

High-quality camera lenses.

Professional audio equipment.

The Ring should accelerate naturally.

The Ring should decelerate naturally.

Rotation should never stop abruptly.

Visitors should feel mechanical precision.

Not software animation.

---

# 10. Selected State

At every moment, only one action is selected.

The selected action should always be the visual focus.

Selection may be communicated through:

Scale.

Brightness.

Opacity.

Depth.

Motion.

Avoid using obvious borders or colorful highlights.

Selection should feel elegant rather than explicit.

---

# 11. Camera Behaviour

The visitor should feel like looking into a three-dimensional object.

The camera should remain stable.

Avoid unnecessary camera movement.

Depth should come from the Ring itself.

Not from excessive camera animation.

Transitions between worlds should feel like moving forward into space.

Never like opening another webpage.

---

# 12. World Transition

Selecting an action should begin a transition.

The transition should feel cinematic.

The Ring gradually moves away.

The selected world becomes larger.

Background continuity should always remain.

Visitors should always know they are still inside the same product.

World transitions should never feel like loading a new website.

---

# 13. Typography

Typography should remain secondary.

The homepage should never be text-heavy.

The largest text should always be:

YIZHEN ZHOU

Supporting text should be minimal.

Avoid subtitles unless absolutely necessary.

Words should support the interaction.

Never replace it.

---

# 14. Ambient Effects

Ambient effects exist only to create atmosphere.

Examples include:

Soft lighting.

Subtle grain.

Very slow particle movement.

Tiny background dots.

No effect should become the visual focus.

If visitors notice the effect before the Ring, it is too strong.

---

# 15. Mobile Experience

The mobile homepage should preserve the same philosophy as the desktop experience.

Interaction should remain intuitive.

The Ring should remain the center of the experience.

The primary interaction changes from mouse wheel to touch gestures.

Users rotate the Ring by swiping horizontally or vertically.

Momentum should remain consistent with the desktop version.

Touch interaction should feel precise.

Not slippery.

Not overly sensitive.

The homepage should never become a simplified menu on mobile.

The experience should remain immersive.

---

# 16. Responsive Behaviour

The homepage should adapt naturally to different screen sizes.

The design language should never change.

Only proportions should change.

Desktop

Large Ring.

More surrounding space.

Tablet

Slightly smaller Ring.

Maintain the same interaction.

Mobile

Compact Ring.

Optimised touch targets.

Minimal interface.

At every screen size, the Ring remains the visual center.

---

# 17. Performance Requirements

Performance is part of the design.

Smoothness is not optional.

Target frame rate:

60 FPS minimum.

Loading should feel instant.

Avoid unnecessary JavaScript.

Avoid heavy assets.

Avoid excessive rendering.

The homepage should remain responsive on modern laptops and mobile devices.

Performance should always be prioritised over visual complexity.

---

# 18. Accessibility

The homepage should remain usable without relying entirely on motion.

Navigation should support keyboard input.

Every interactive element should be reachable.

Animations should respect reduced-motion preferences.

Text should remain readable under different display conditions.

Accessibility should improve the experience.

It should never compromise the design language.

---

# 19. Future Expansion

The homepage is expected to evolve.

Future versions may include:

Interactive introductions.

Seasonal visual changes.

AI-generated ambient elements.

Context-aware greetings.

Additional Ring behaviours.

These features should never replace the core interaction.

The Ring remains the identity of the homepage.

Everything else is secondary.

---

# 20. Acceptance Criteria

The homepage is considered complete only when all of the following are true.

Visitors immediately recognise that the homepage is interactive.

The Ring feels mechanically precise.

Scrolling controls the Ring naturally.

Transitions feel cinematic.

The background creates atmosphere without distraction.

Typography remains secondary.

The homepage feels calm.

The homepage feels premium.

The homepage performs smoothly.

The homepage works consistently across desktop and mobile.

Most importantly,

Visitors should leave the homepage remembering the experience rather than the interface.

If the interaction feels memorable but the design feels effortless, the homepage has succeeded.

End of HOME_PRD.
