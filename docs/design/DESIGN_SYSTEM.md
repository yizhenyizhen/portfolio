# DESIGN_SYSTEM.md

---

# 1. Purpose

This document defines the visual language of the website.

Every component should follow these rules.

Consistency is more important than creativity.

If a future design conflicts with this document, redesign it before implementation.

---

# 2. Design Philosophy

The interface should disappear.

Content becomes the focus.

Interaction becomes the memory.

The website should feel calm before it feels beautiful.

Premium before impressive.

Timeless before trendy.

Precision before decoration.

---

# 3. Color System

Primary Background

Pure Black

#000000

---

Secondary Background

Very Dark Gray

Used only when depth is required.

Never use multiple gray backgrounds on one screen.

---

Primary Text

Pure White

Highest contrast.

Used for titles.

---

Secondary Text

Soft Gray

Used for descriptions.

Should never compete with titles.

---

Accent Color

Reserved.

No permanent accent color is defined in Version 1.

The accent color will be introduced only after the homepage has reached visual maturity.

Accent colors should communicate state.

Never decoration.

---

# 4. Typography

Titles

Large.

Bold.

Confident.

Minimal.

Maximum two lines.

---

Body Text

Short.

Clear.

One idea per paragraph.

Avoid long descriptions.

---

Numbers

Numbers should feel clean.

Large numbers may become visual elements.

---

Capitalization

Use sentence case whenever possible.

Avoid excessive uppercase.

YIZHEN ZHOU is the only permanent exception.

---

# 5. Spacing

Spacing creates hierarchy.

Never reduce spacing to fit more content.

Use generous whitespace.

Every screen should breathe.

Visual rhythm is more important than density.

---

# 6. Corners

Avoid excessive rounded corners.

The website should feel engineered.

Not playful.

Rounded corners should remain subtle.

---

# 7. Shadows

Shadows should communicate depth.

Not decoration.

Avoid large blurry shadows.

Prefer lighting over shadow.

---

# 8. Motion

Motion should feel physical.

Never elastic.

Never bouncy.

Never exaggerated.

Acceleration feels natural.

Deceleration feels controlled.

Objects carry weight.

Objects possess inertia.

Motion should feel closer to industrial products than mobile games.

---

# 9. Depth

Depth comes from lighting.

Not from blur.

Not from decoration.

The interface should feel layered.

Not stacked.

---

# 10. Components

Every component should follow the same visual language.

Navigation.

Buttons.

Cards.

Dialogs.

Future components should feel like they belong to one family.

---

# 11. Icons

Icons should remain minimal.

Avoid decorative icons.

Every icon should communicate function immediately.

Prefer line icons.

Avoid colorful illustrations.

---

# 12. Images

Images should be cinematic.

Sharp.

High resolution.

Strong composition.

Never use generic stock photography.

Images should communicate atmosphere before information.

---

# 13. Animation Duration

Fast Interaction

150–250ms

Standard Transition

300–500ms

Large Scene Transition

600–1000ms

Never animate longer without purpose.

---

# 14. Performance

Every animation should remain smooth.

Target:

60 FPS

Interaction latency should feel immediate.

Performance always wins over visual effects.

---

# 15. Future Evolution

This Design System is expected to evolve.

New rules should extend the existing language.

Never replace it.

Consistency across years is more valuable than temporary visual trends.

End of Design System.
